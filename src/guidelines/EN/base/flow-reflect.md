# Base Flow Template: Reflect

**Phase**: reflect
**Flow ID**: base-flow-reflect
**Version**: 1.0.0
**Description**: Comprehensive reflection phase that captures lessons learned, analyzes performance, improves processes, and prepares knowledge for future projects.

## Overview

This flow template defines the systematic process for reflecting on project outcomes, capturing lessons learned, analyzing performance metrics, improving processes, and preserving organizational knowledge.

## Entry Criteria

- [ ] Release phase completed successfully
- [ ] System stable in production for defined period
- [ ] Post-release monitoring data available
- [ ] User feedback collected and analyzed
- [ ] Performance metrics gathered
- [ ] Team available for reflection activities

## Exit Criteria

- [ ] Comprehensive retrospective completed
- [ ] Lessons learned documented and shared
- [ ] Performance analysis completed
- [ ] Process improvements identified and planned
- [ ] Knowledge artifacts created and stored
- [ ] Team feedback collected and addressed
- [ ] Future improvements planned

## Success Metrics

- **Retrospective Coverage**: 100% of project phases reviewed
- **Lesson Capture**: ≥95% of key lessons documented
- **Improvement Identification**: ≥10 actionable improvements identified
- **Knowledge Preservation**: All critical knowledge preserved
- **Team Satisfaction**: ≥90% team satisfaction with reflection process
- **Process Improvement**: ≥80% of identified improvements implemented

## Reflection Frameworks & Methods

### Start-Stop-Continue
- **Start**: Things to start doing in future projects
- **Stop**: Things to stop doing that didn't work
- **Continue**: Things that worked well and should continue

### 4Ls (Liked, Learned, Lacked, Longed For)
- **Liked**: What went well and team enjoyed
- **Learned**: New knowledge and insights gained
- **Lacked**: What was missing or needed
- **Longed For**: What team wished they had

### Sailboat Exercise
- **Wind**: Things that helped push the project forward
- **Anchor**: Things that held the project back
- **Rocks**: Potential risks and obstacles ahead
- **Island**: Goal or destination

### Timeline Retrospective
- **Positive Events**: Key successes and achievements
- **Negative Events**: Challenges and issues faced
- **Learning Points**: Insights and lessons learned
- **Future Actions**: Improvements for next time

- **Lessons Identification**: Comprehensive analysis, synthesis skills
- **Improvement Planning**: Clear priorities, resource availability
- **Knowledge Preservation**: Documentation systems, sharing platforms
- **Reflection Closure**: Team participation, closure activities

## Risk Mitigation

| Risk | Probability | Impact | Mitigation Strategy |
|------|-------------|--------|-------------------|
| Incomplete data | Medium | Medium | Early data collection, multiple sources |
| Team participation | Low | High | Engaging facilitation, safe environment |
| Lessons not implemented | High | High | Action planning, ownership assignment |
| Knowledge lost | Medium | High | Documentation, sharing processes |
| Reflection too negative | Low | Medium | Balanced approach, focus on learning |
